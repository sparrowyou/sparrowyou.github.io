处理多返回值

  1. 结构体
    #include <iostream>
 #include <fstream>
 #include <string>
 #include <sstream>

 struct ShaderProgramSource
 {
     std::string VertexSource;
     std::string FragmentSource;
 };

 static ShaderProgramSource ParseShader(const std::string& filepath)
 {
     std::ifstream stream(filepath);
     
     enum class ShaderType
     {
         NONE = -1, VERTEX = 0, FRAGMENT = 1
     };

     std::string line;
     std::stringstream ss[2];
     ShaderType type = ShaderType::NONE;

     while(getline(stream, line))
     {
         if(line.find("#shader") != std::string::npos)
         {
             if(line.find("vertex") != std::string::npos)
                 type = ShaderType::VERTEX;
             else if (line.find("fragment") != std::string::npos)
                 type = ShaderType::FRAGMENT;
         }
         else
         {
             ss[(int)type] << line << "\n";
         }
     }

     return {ss[0].str(), ss[1].str()};
 }

const 关键字

const 是C++中的一个关键字，用于指定变量、成员函数、函数参数等的属性。它表示“常量”或“不可变”，具体作用取决于它所修饰的对象。以下是 const 关键字的不同用法和含义：

  1. 常量变量：如果你将一个变量声明为 const，则该变量的值不能被修改。这意味着它是只读的。例如：

    const int myConstant = 42;
myConstant = 5; // 这将导致编译错误，因为myConstant是常量，不可修改

  2. 常量指针：const 也可以用于指针，表示指针指向的数据不能被修改。例如：

    const int* ptr = &myConstant;
*ptr = 5; // 这将导致编译错误，因为ptr指向的数据是常量

  3. 指向常量的指针：在这种情况下，指针本身是可变的，但它指向的数据是不可变的。例如：

    int anotherConstant = 10;
int* const constantPtr = &anotherConstant;
*constantPtr = 5; // 合法，可以修改anotherConstant的值
constantPtr = &myConstant; // 这将导致编译错误，因为constantPtr本身是常量

  4. 常量成员函数：在类中，成员函数可以声明为 const，这表示它们不会修改类的成员变量。这使得编译器能够在常量对象上调用这些函数。例如：

    class MyClass {
public:
    void ModifyData() { /* 修改成员变量 */ }
    void AccessData() const { /* 不修改成员变量 */ }
};

const MyClass myObj;
myObj.ModifyData(); // 错误，不能在常量对象上调用修改成员的函数
myObj.AccessData(); // 合法，可以在常量对象上调用不修改成员的函数

  5. 常量引用参数：const 也可以用于函数参数，表示函数不会修改传递给它的参数。这使得函数可以接受常量对象作为参数。例如：

    void ProcessData(const int& data) {
    // data 是常量，不能被修改
}

int myValue = 42;
const int myConstant = 5;
ProcessData(myValue); // 合法，myValue作为参数不会被修改
ProcessData(myConstant); // 合法，myConstant作为参数不会被修改

类成员变量初始化列表

类成员初始化列表（Class Member Initialization List）是C++中用于初始化类的成员变量的一种方式。通常，成员变量在类的构造函数中进行初始化，但初始化列表提供了一种更有效和直接的方式来初始化成员变量，特别是对于复杂类型的成员变量。

初始化列表以冒号:开始，后面跟随成员变量的初始化。以下是一个示例，演示了如何在类的构造函数中使用成员初始化列表：

class MyClass {
private:
    int intValue;
    double doubleValue;

public:
    MyClass(int a, double b) : intValue(a), doubleValue(b) {
        // 构造函数体
        // 在这里可以执行其他初始化操作
    }

    void PrintValues() {
        std::cout << "intValue: " << intValue << ", doubleValue: " << doubleValue << std::endl;
    }
};

int main() {
    MyClass obj(42, 3.14); // 使用初始化列表初始化成员变量
    obj.PrintValues();
    return 0;
}

在这个示例中，MyClass 类有两个私有成员变量 intValue 和 doubleValue。构造函数使用成员初始化列表来初始化这两个成员变量，而不是在构造函数体中执行初始化。这提供了更高效的初始化方式，特别是对于复杂类型的成员变量。

使用成员初始化列表的优点包括：

  1. 效率: 初始化列表允许在对象构造时直接初始化成员变量，而不需要额外的赋值操作。这可以减少不必要的临时对象的创建和销毁。

  2. 确保初始化: 初始化列表确保成员变量在进入构造函数体之前得到了初始化。这对于某些成员变量的初始化很重要，特别是对于const成员变量或引用成员变量。

  3. 直接初始化: 初始化列表支持直接初始化语法，使代码更清晰和紧凑。
